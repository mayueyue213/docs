# render

根据 Watcher 中的说明，Watcher实例化的过程会主动调用updateComponent

updateComponent = () => {
  vm._update(vm._render(), hydrating)
}

updateComponent会调用vm._update，其第一个参数是vm._render，这就是本文研究的内容。

# vm._render

_render函数定义在 src/core/instance/render.js

Vue.prototype._render = function (): VNode {
  const vm: Component = this
  const { render, _parentVnode } = vm.$options

  setSlots(); // ...

  vm.$vnode = _parentVnode
  // render self
  let vnode
  try {
    currentRenderingInstance = vm
    vnode = render.call(vm._renderProxy, vm.$createElement)
  } catch (e) {
    handleRenderError(); // ...
  } finally {
    currentRenderingInstance = null
  }

  if (Array.isArray(vnode) && vnode.length === 1) {
    vnode = vnode[0]
  }

  if (!(vnode instanceof VNode)) {
    vnode = createEmptyVNode()
  }

  vnode.parent = _parentVnode
  return vnode
}

_render是原型上的一个方法，因此vm指向Vue实例，我们重点关注下面这个调用

vnode = render.call(vm._renderProxy, vm.$createElement)

这个options.render就是我们在编译过程中最后通过compileToFunctions返回的，如果以下方代码为例

const app = new Vue({
  template: '<div class="header">hello</div>'
})

app.$mount('#app')

它是这样的结果

function anonymous() {
  with(this){return _c('span',[_v("hello")])}
}

在生产环境下，_renderProxy 就是vm，这里使用了with绑定了上下文为vm，因此这里的_c，_v是vm实例的属性。执行之后返回的vnode如下

我们要继续深入，需要了解这些_c，_v这些内容，_c其实是最知名的$createElement方法，而_v是createTextVNode，这些实例属性我们在后面单独讲解。

关于 render 执行，至此我们只要知道返回的是一个 VNode 即可，从中我们也可以看到，里面返回的 VNode，不是单个节点，已经通过 children 和 parent 维护了层次关系。

# 总结

render这个词，实在是有太多的含义了。在这里当它是一个名词时，说的是编译阶段生成的 render函数，当描述的是一个动作时，表示的是调用render函数生成虚拟 DOM 的过程。

如果只能记住一句话 —— render 函数的作用是中间代码生成虚拟DOM

至此，从 template 开始，经历了以下流程

template(字符串) -> parse(生成AST) -> optimize(AST优化) -> codegen(中间代码生成) -> compileToFunctions(中间代码转化为render函数) -> render(生成VDOM Tree)
