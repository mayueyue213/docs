# placeholder node

在源码中，placeholder node 表示子组件使用时，在 VDOM 树中用来表示这个组件的那个VNode节点(这里为方便称为 cnode)，这个 cnode 它代表的是这里有一个xxx组件，但是它本身并不是组件自身的定义。

本文通过例子说明 placeholder node 究竟是怎样一个概念，为方便本文后续称为 占位节点。

# 示例

const Demo = {
  template: '<span>demo</span>'
}

const App = {
  components: {
    Demo
  },

  template: `<div class="header">
    <demo></demo>
  </div>`,
  el: '#app',
};

new Vue(App)

# 理解占位节点

为什么组件对应的节点被称为占位节点呢？看一下上面示例生成的 VDOM Tree 结果吧

{
  asyncFactory: undefined
  asyncMeta: undefined
  children: [
    {
      asyncFactory: undefined
      asyncMeta: undefined
      children: undefined
      componentInstance: undefined
      componentOptions: {propsData: undefined, listeners: undefined, tag: "demo", children: undefined, Ctor: ƒ}
      context: // 根实例
      data: {on: undefined, hook: {…}}
      elm: undefined
      fnContext: undefined
      fnOptions: undefined
      fnScopeId: undefined
      isAsyncPlaceholder: false
      isCloned: false
      isComment: false
      isOnce: false
      isRootInsert: true
      isStatic: false
      key: undefined
      ns: undefined
      parent: undefined
      raw: false
      tag: "vue-component-1-demo"
      text: undefined
    }
  ]
  componentInstance: undefined
  componentOptions: undefined
  context: // 根实例
  data: {staticClass: "header"}
  elm: undefined
  fnContext: undefined
  fnOptions: undefined
  fnScopeId: undefined
  isAsyncPlaceholder: false
  isCloned: false
  isComment: false
  isOnce: false
  isRootInsert: true
  isStatic: false
  key: undefined
  ns: undefined
  parent: undefined
  raw: false
  tag: "div"
  text: undefined
}

这是 App 组件的 vnode，这里 children[0](demoVnode)，它表示的就是我们的 Demo 组件，或者更准确来说，表示的是下面的 <demo></demo> 标签，它表示的是在被使用的地方占个位置

<div class="header">
  <demo></demo>
</div>

在这里，表示 Demo 组件的children[0]子节点，它只是表示”这里有一个demo组件“，而不是 ”demo 组件本身的定义“，这里 demoVnode 的 tag 是 vue-component-1-demo，但 Demo 组件实际上是个 span，这里只是占了个位置，还没坐上去呢，这就是为什么表示组件的 vnode 节点被称为 ”占位节点“。

注：至于 Demo 组件本身的定义，在创建 VDOM Tree 的过程中，对于组件会构造 Vue 的子类构造函数，保存在了 demoVnode.componentOptions.Ctor 属性上，在后面的流程中会根据这个 Ctor 去生成组件对应的 Vue 实例和 DOM 元素，保存到这个 demoVnode 上。
