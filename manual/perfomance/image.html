# 图片

让图片更小，让图片更少。

# 更小 - 格式选择

可以参考 某东 (opens new window) 对应看一下，这类图片很多的网站，在图片的选择上是比较恰当的。

格式 透明度 适用场景
JPEG 否 色彩丰富，大图，照片，不规则图
PNG 是 高保真，色彩单一，透明，形状类考虑 SVG
WebP 是 色彩不太丰富的图，图形和半透明图
SVG 是 高保真要求，缩放需求，icon，各种形状
GIF 是 动图，适合小图，当内容大时考虑 video 替代

在保证兼容性，显示效果的基础上，尽可能降低数据量，WebP 与 SVG 的更多使用，会是一个趋势。

  • PNG：无损压缩，保存细节，适用有精细要求的场景，同时注意文件较大。
  • 动画可以考虑 GIF, SVG, WebP, video 等多种方案。
  • WebP：兼容性相对较差，可以做支持性检测，利用图片服务器做处理转换，如果是可控客户端，WebP 优先。

WebP 的探测加载，已经成为比较通用的方案，某东就是加载了一张图片来判断是否支持，然后再将一些图片添加.webp标记交给图片服务器处理。

# 更小 - 图片压缩

基于 imagemin (opens new window) ，可以使用 Webpack 插件 ImageMinimizerWebpackPlugin (opens new window)，也可以定制小工具去处理。需要注意

  • 压缩与生产环境构建分离
  • 避免重复压缩

自然不需要在每次构建时进行压缩，在构建之外提前执行压缩一遍即可。结合团队的工程结构设计，应用场景对压缩参数的要求等，可以再封装一个命令行小工具，进一步减少配置量，甚至做到“免配置“一键执行。

其他工具:

  • 客户端：PPDuck3 (opens new window)，腾讯 智图 (opens new window)
  • 线上工具：tinypng (opens new window)
  • 命令行: ImageMagick convert
  • UI 工具：有些 UI 工具已经自带了较好的压缩能力，导出可直接使用

看团队项目情况和具体业务需要确定一套处理方式吧，反正尽量减少工作量。

# 更少请求 - 懒加载

通过 js 判断出当前图片是否进入或者即将进入到可视区再进行图片加载。主要涉及到加载时机判断和加载完成之前的图片占位问题。

# 加载条件触发

也就是图片所在位置与视口相对位置的判断，判断进入视口有两种方式

  • 监听滚动等事件，或者通过其他方式周期性检查所在位置，如通过 getBoundingClientRect 获取相对视口的位置。vue-lazyload (opens new window) 就是这么做的。
  • 利用 IntersectionObserver 这个 API 来设置回调，在回调中加载。 lazyload (opens new window) 就是这样做的。

不过IntersectionObserver虽然好用，但支持性仍不足，需要 polyfill。

# 占位图

懒加载相关的一个关键内容是占位图的应用。图片尚未加载的情况下，需要展示一个占位，占位常有以下实现

  • 结构背景图占位：像文章一类的结构特点比较鲜明的内容，可以用占位图来提升视觉体验，比如掘金 (opens new window)的文章，加载时可以看到很明显的如下占位图。
  • 背景色占位：使用背景色占满图片空间
  • 小背景图铺满：使用一个 极小的纯色图片 (opens new window)，通过 repeat 来铺满整个图片空间

某东的占位图，就综合使用了 “背景色占位” 和 ”小背景图铺满“ 的占位方式

# CSS 中的懒加载

在 CSS 中，无论是 背景图片，还是字体资源，如果从未使用到，就不会下载。

/* 顺便提一下字体，如果没有元素对应的样式使用了 "Open Sans"，对应字体文件不会加载。 */
@font-face {
  font-family: "Open Sans";
  src: url("/fonts/OpenSans-Regular-webfont.woff2") format("woff2"),
       url("/fonts/OpenSans-Regular-webfont.woff") format("woff");
}

.demo {
  background: url("img/blank.jpg")
}

.demo {
  background: url("img/hero.jpg")

# 更少请求 - Web Font 替代

使用 iconfont (opens new window) 等矢量图标库来替代一些 icon 图片，比如说某东，某猫等都使用了 iconfont。

# 其他

还有其他的一些图片相关的方案

  • 渐进式 JPEG: 按照其扫描模式，从理论上来说能够提升用户体验，但是其实消耗性能，需要综合考虑
  • LQIP/SQIP: 使用LQIP (opens new window) 和 SQIP (opens new window) 生成低质量的，也可能提升用户体验，但是同时带来额外的加载量。
  • 雪碧图：我已经舍弃了。雪碧图需要配合 CSS 使用，对 CSS 的依赖过强，且调整并不灵活，Web Fonts，SVG 等都是更灵活的替代方案。
  • 删除图像元数据：适合大量照片展示类的内容
  • 根据网络情况来加载不同的图片

# 总结与参考

图片相关的方案很多，考虑是否必要用图片，是否应该用 Web Fonts 或者 CSS 替代，在需要图片时，可以优先考虑 WebP 和 SVG。
