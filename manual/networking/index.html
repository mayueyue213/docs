# 应用层

应用层主要的协议有 HTTP，DNS，DHCP，SSH 等

# HTTP

HTTP 需要结合其几个历史版本的发展进行了解，了解每个版本增加了一些什么内容。

  • HTTP
    • HTTP/0.9
      • 单行协议
    • HTTP/1.0
      • 支持多种文件
        • 增加 header: 客户端请求协商，服务端资源描述
          • Status Code 表示协商结果
    • HTTP/1.1
      • 长连接
        • 队头阻塞
          • Chrome 单域名 6 个并发上限
            • CDN 多域名
      • 虚拟主机多域名
        • Host
      • Chunk Transfer
      • Cookie
    • HTTP/2
      • 多路复用：二进制分帧层拆分 HTTP 包，交错传输，客户端设置优先级请求，服务端优先级处理返回，客户端拼接返回。
    • HTTP/3
      • QUIC：基于 UDP，替换 HTTP 下层协议 TCP

# DNS

DNS 用于根据域名获取 IP 地址，需要了解常规的 DNS，以及 CDN DNS 和 HTTP DNS，还需要了解 SLB。

  • HttpDNS：传统的 DNS 解析，在数据缓存更新，域名转发，域名更新(如机房切换)等存在一些问题。HTTP DNS，就是不走传统 DNS，自己搭建基于 HTTP 的 DNS 服务器集群。一般 HttpDNS 用于手机端应用，使用 SDK 加载缓存 HttpDNS Server 的 IP 列表，通过 HTTP 绕过传统 DNS 获得 IP。

  • CdnDNS: CDN 一般会在权威 DNS 服务器中设置 CNAME，当访问某个域名，返回的不再是源站的 IP 地址，而是返回一个 CDN 域名，本地 DNS 服务器拿到这个域名之后，去访问 CDN 自己搭建的权威 DNS 服务器，这个 DNS 服务器再次返回负载均衡器的 CNAME，本地 DNS 再次请求负载均衡器即可得到缓存服务器的 IP。在获取到缓存服务器的 IP 之后，再依次访问边缘节点，区域节点，中心节点，最后回源。

# DHCP

动态主机协议，用以在局域网中设备接入分配 IP，基本的原理是接入设备和 DHCP Server 通过广播进行通信协商，协商好之后通过广播广而告之。

# 传输层

在传输层与应用层间通过套接字进行连接，端口号作为套接字的标识，通过端口号实现多路复用和多路分解，从而实现数据正确交付到不同的应用。

# TCP

TCP 是面向连接的服务，保证数据的可靠交付。

  • 可靠数据传输
    • 回退 N 步：还有未确认数据，但发送方的定时器超时都未收到任何窗口内数据包的确认，重传 N 个
    • 选择重传：只重传丢失的分组
    • 滑动窗口：允许的已发送未被确认分组数大小，这是流量控制的一种方式
  • 面向连接：常说的三次握手。连接就是双方先简单沟通一下，各自建立一定的数据结构来维护双方交互的状态。
  • 拥塞控制
    • 拥塞窗口：用以限制发送流量的速率
    • 快速重传：连续收到 3 个相同序号的 ACK
      • 冗余ACK：下一个待确认分组的响应 ACK
    • 慢启动

# UDP

UDP 用户数据包，除了多路复用和多路分解，以及少量的差错检测(校验和)，几乎没有对 IP 增加别的东西。 UDP 适用于一些对于交付可靠性要求不那么高的程序，通过 UDP 能够降低时延，减少资源消耗。

  • 无连接
  • 无状态
  • 不保证可靠交付

UDP 相对与 TCP，应用层程序能做的控制更多，很多交互，重传相关的事情可以在应用层去选择策略去做，QUIC 就是使用 UDP，将一部分类似 TCP 功能的实现交给了应用层去实现。

DNS 选择 UDP，则是因为其不需要建立连接，可以尽量减少时延，而可靠交付，对于 DNS 并不那么重要。

# 网络层

网络层主要关注三个内容，IP，ICMP 和 路由器。

IPv6 相比于 IPv4

  • IPv6 地址空间更大，128
  • 4 是变长头部，6 固定 40 字节
  • 兼容性，双栈技术，隧道技术和 NAT-PT(Network Address Translation - Protocol Translation) 支持
  • 强制集成 IPsec，提升安全性

# 链路层

链路，是指连接相邻结点的通信信道，链路层要解决的是共享信道上消息传输的问题，基本问题有

  • 发给谁？
  • 出错了怎么办？
  • 大家都要用信道怎么办？

由此产生了链路层相关的多种协议。

# 发给谁

解决这个问题对应的处理叫做 成帧，梦想成帧的这个成帧

成帧：接收到上层的数据包之后，进一步封装，添加链路层目标地址，源地址进行标识

图片占位

链路层使用 6 个字节的地址标识设备，由于链路层的主要功能是解决媒体访问控制(Media Access Control)的问题，这个地址称为 MAC 地址，叫做链路层地址，物理地址更合适也行。

这里有一个前提，需要知道目标 MAC 地址，这涉及到的是 ARP 协议。

# 出错了怎么办

对应的链路层基本服务叫做 差错检测

差错检测：校验传输过程中的差错，部分方案提供纠错能力

常见的差错检测方法有

  • 奇偶校验：补充一个校验位，奇校验是要使得 1 的次数为奇数，偶校验是要使得 1 的次数为偶数。
  • 检验和
  • CRC(Cyclic Redundance Check，循环冗余校验)

其中链路层帧使用的是 CRC。

# 大家都要用信道怎么办

对应的基本服务叫做 多路访问，共享信道上，大家都要用信道，谁先用，怎么用的问题，跟日常的道路交通问题类似。多路访问协议有

  • 信道划分协议：划分为多条车道，各用各的
    • 时分多路复用(TDM，Time-Division Multiplexing)：为接入结点分配时间帧
    • 频分多路复用(FDM，Frequency-Division Multiplexing)：为节点分配不同频率
    • 码分多址(CDMA)
  • 随机接入协议：先上路再说，撞到南墙(信道拥堵)再回头，好了伤疤继续(一定延迟后重新发送)
    • 载波侦听多路访问(CSMA)
      • 载波侦听：说话之前先听，看看有没有人在发言
    • 具有碰撞检测的 CSMA(CSMA/CD)
      • 载波侦听
      • 碰撞检测
  • 轮流协议
    • 轮询协议
    • 令牌协议

诸多技术方案，大概了解，关注 CSMA/CD，这是以太网用的。

# 链路层设备

由于两个设备之间，并不是直接的线路连接的，设备间的消息传输需要进行中转。涉及的设备是集线器和交换机。

  • 集线器：将消息广播给所有的接入方
  • 交换机：收集每个口接入的 MAC 地址，根据 MAC 地址进行转发

由于在链路层，这一层的一些设备也称为所谓的二层设备，所以有二层交换机这种说法。

# 物理层

物理层就是真实的物理设备，比如网线，集线器(Hub)，交换机等。

# Q & A

  • SSL 是综合使用了对称加密和非对称加密，非对称加密用于协商生成秘钥，对称加密用于数据传输的加解密。

  • 客户端传输一个随机数 client_random 和支持的加密方法列表到 server
  • 服务器传给客户端 server_random，加密方法列表和数字证书 给浏览器
  • 浏览器验证公钥，生成 pre_random，用公钥加密传给服务器
  • 服务器用 cilent_random，server_random 和 pre_random 使用公钥加密生成 secret
  • 之后的数据传输，还是用 secret 作为秘钥，进行对称加密的解密

  • GET,POST: 最常用的接口请求
  • OPTIONS: 常见与 CORS 的所谓复杂请求中的预检请求
