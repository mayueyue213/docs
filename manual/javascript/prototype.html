# new实例化发生了什么

看下面的例子

function sayName() {
  console.log(this.name);
};
function Person(name, age, job) {
  this.name = name;
  this.age = age;
  this.job = job;
  this.sayName = sayName;
}

Person.prototype.toUpperCaseName = function() {
  return (this.name || '').toUpperCase();
};

const person1 = new Person("Nicholas", 29, "Software Engineer");
const person2 = {};
Object.setPrototypeOf(person2, Person.prototype);
person2.name = "Nicholas";
person2.age = 29;
person2.job = "Software Engineer";
person2.sayName = sayName;

问题的一种近似表述 —— 对象字面量person2如何转换成跟person1一样？ ”一样“的意思就是除了在内存中放的位置不同，其他都一样，从person2的设置上看有这么三个步骤：

  1. 创建一个空对象；
  2. 设置[[Prototype]]属性；
  3. 设置属性和方法。

然后再看Person构造函数执行的过程，与person2的设置过程不完全一样。在Person函数中设置属性是this.xxx，this执行当前实例对象，而对于person2就是person2.xxx，也就是说有一个设置this指向的步骤。对于person1是函数的返回值，这意味着最终有一个return的步骤。最终new过程 5 个步骤如下：

  1. 在内存中创建一个新对象。
  2. 这个新对象内部的[[Prototype]]特性被赋值为构造函数的prototype属性。
  3. 构造函数内部的this被赋值为这个新对象（即this指向新对象）。
  4. 执行构造函数内部的代码（给新对象添加属性）。
  5. 如果构造函数返回非空对象，则返回该对象；否则，返回刚创建的新对象。
