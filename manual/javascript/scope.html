# 作用域

作用域是变量的可访问范围，是一套存储和查找变量的规则。

  • 存储：变量归属于谁。定义的这个变量，放在谁家，归谁管，这对应作用域的分类问题。JavaScript 中有全局作用域、块级作用域和函数作用域。
  • 查找：我到哪里去找我要的变量？如何在不同的归属地之间查找的规则，这对应作用域链相关的内容。

# 静态作用域

在描述 JS 的几类作用域之前，先介绍一些静态作用域和动态作用域的概念。 静态作用域 也叫 词法作用域，与之相对的是动态作用域。JavaScript 使用的是词法作用域，词法作用域关注在 何处声明，查找的时候，由声明的位置开始去查找标识符

var value = 12;
function foo() {
  console.log(value); // 12
}

function bar() {
  var value = 24;
  foo();
}

bar();
foo();

# 动态作用域

bash 使用的是动态作用域，有的文章会说

动态作用域关注的是在何处调用

也就是说，如果在 foo 函数中未找到 value 标识符，则向 foo 函数调用的执行环境中查找。

这和静态作用域的标识符查找过程形成了鲜明对比，但是并不准确。看这个 bash 的例子

#!/bin/bash

value=12;
foo(){
  echo "$value";
}

bar() {
  value=24;
  foo
}

foo
bar
foo

对于动态作用域，可以理解为它只有一个全局的变量环境，声明的所有标识符都存在一个对象上。不管在哪里修改value，都是改的同一个变量，它没有局部作用域的概念。在静态作用域中，在函数中var value = 24是声明一个局部的变量，而在动态作用域中，value=24只是一个重新赋值而已。

# 块级作用域

块级作用域(block scope)，就是在{}中形成的一个独立的词法空间，是一个比函数更小级别的局部变量空间。函数作用域对应有函数执行上下文，而块级作用域并没有块级执行上下文。

块级作用域在执行上下文中表现为词法环境栈中的一个 词法环境对象

function foo() {
  var a = 1;
  let b = 2;
  {
    const c = 3;
    var d = 4;
    let e = 5; 
  }
}

foo();

# 函数作用域

函数作用域的理解，关键还是在于理解执行上下文

  • 变量提升，是提升到函数级别，提升到执行上下文中的变量环境
  • 函数执行时有独立的执行上下文

从执行上下文的角度看，函数作用域，只是其一部分，是其上变量存储结构和查找规则。

  • 存储结构：变量环境，词法环境，词法环境栈
  • 查找规则：执行上下文中，从词法环境栈顶，到变量环境

# 总结

作用域是变量的可访问范围，是一套存储和查找变量的规则。作用域链就是根据静态作用域标识符查找的规则，去查找变量时，能够经过的一个个作用域，根据先后顺序串起来形成的链。

JavaScript 采用的是静态作用域，每个词法作用域都可以理解为以下这样一个对象

{
  // ...
  [[outer]]: // parent scope reference
}
